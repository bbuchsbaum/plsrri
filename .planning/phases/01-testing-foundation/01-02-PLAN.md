**mod_analyze.R elements to tag:**
- Progress bar container: `data-test = "analyze-progress"`
- Progress percentage text: `data-test = "analyze-progress-percent"`
- Stage display: `data-test = "analyze-stage"`
- Elapsed time: `data-test = "analyze-elapsed"`
- Cancel button: `data-test = "analyze-cancel-btn"`
- Back button: `data-test = "analyze-back-btn"`
- Error display: `data-test = "analyze-error"`
- Completion display: `data-test = "analyze-complete"`
- Explore button (on completion): `data-test = "analyze-explore-btn"`

**Implementation:** Wrap elements in div() with the attribute, or add directly to actionButton/input calls using the `...` argument or tags$div wrapper.

Example:
```r

actionButton(ns("btn_continue"), "Continue")

# After
actionButton(ns("btn_continue"), "Continue", `data-test` = "setup-continue-btn")

# For inputs without direct attribute support, wrap:
div(`data-test` = "setup-num-conditions",
  numericInput(ns("num_conditions"), ...)
)
```
