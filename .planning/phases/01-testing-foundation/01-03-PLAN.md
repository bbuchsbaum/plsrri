**Testing challenge:** The analyze module calls plsrri::run() which is slow. For unit tests:
- Mock the analysis or skip actual execution
- Focus on state machine logic (ready -> running -> complete/error)
- Test button handlers without triggering full analysis

**Approach:** Create a minimal spec that will complete quickly:
```r
# Use a tiny spec for testing
tiny_spec <- make_mock_spec(n_groups = 1, n_subj = 3, n_cond = 2, n_voxels = 10)
tiny_spec$num_perm <- 0
tiny_spec$num_boot <- 0
```

Or mock the run function in tests:
```r
with_mocked_bindings(

    # test code
  }
)
```
