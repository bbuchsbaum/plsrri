  <name>Task 1: Create surface mapping infrastructure</name>
  <files>inst/shiny/R/fct_surface_mapper.R, tests/testthat/test-fct_surface_mapper.R</files>
  <action>
Create fct_surface_mapper.R with pure functions for volume-to-surface mapping:

1. **get_fsaverage_surfaces(geometry = "inflated")**
   - Wraps neurosurf::load_fsaverage_std8(geometry)
   - Returns list(lh = surface, rh = surface)
   - Validates geometry parameter (pial, white, inflated, flat)
   - Caches result per geometry type (use environment-based cache)

2. **create_surface_sampler(surfaces, mask, hemisphere = "lh")**
   - Creates vol_to_surf sampler for repeated mapping
   - Uses neurosurf::surface_sampler() with these defaults:
     - knn = 6 (number of neighbors)
     - dthresh = 16 (distance threshold in mm)
     - sampling = "midpoint"
   - Returns sampler object for use with apply_surface_sampler

3. **map_volume_to_surface(sampler, volume, fun = "avg")**
   - Applies pre-computed sampler to a volume
   - Wraps neurosurf::apply_surface_sampler(sampler, volume, fun)
   - Returns mapped surface values
   - Handles missing data (fill = 0 for outside mask)

4. **map_result_to_surfaces(result, lv, what, surfaces, samplers)**
   - High-level function for mapping PLS result to both hemispheres
   - Extracts volume using plsrri::bsr() or plsrri::salience() with as_neurovol=TRUE
   - Applies sampler to each hemisphere
   - Returns list(lh = mapped_surf, rh = mapped_surf)

Create test-fct_surface_mapper.R with unit tests:
- get_fsaverage_surfaces returns list with lh/rh
- get_fsaverage_surfaces validates geometry parameter
- get_fsaverage_surfaces caches results (same object on second call)
- create_surface_sampler returns sampler object (mock neurosurf)
- map_volume_to_surface applies sampler (mock neurosurf)
- map_result_to_surfaces handles bsr correctly
- map_result_to_surfaces handles salience correctly
- Functions handle NULL/missing inputs gracefully

Use mockery or conditional skip when neurosurf not available, similar to existing test patterns.

  <name>Task 2: Add SurfwidgetRenderer to brain renderer abstraction</name>
  <files>inst/shiny/R/fct_brain_renderer.R, tests/testthat/test-fct_brain_renderer.R</files>
  <action>
Extend fct_brain_renderer.R with SurfwidgetRenderer:

1. **Add is_widget() method to BrainRenderer base class**
   - Default implementation returns FALSE
   - Enables polymorphic output type detection

2. **Create SurfwidgetRenderer R6 class**
   - Inherits from BrainRenderer
   - Fields:
     - surfaces: Cached fsaverage surfaces (NULL initially, lazy-loaded)
     - samplers: List of cached samplers keyed by mask hash
     - geometry: Current geometry type (default "inflated")
   - Methods:
     - initialize(geometry = "inflated"): Store geometry, surfaces/samplers start NULL
     - is_widget(): Returns TRUE
     - render(result, lv, what, threshold, ...):
       - Lazy-load surfaces if NULL (get_fsaverage_surfaces)
       - Get or create samplers for result mask
       - Map volume to surfaces (map_result_to_surfaces)
       - Create surfwidget with neurosurf::surfwidget()
       - Pass thresh = c(-threshold, threshold)
       - Return htmlwidget object
     - set_geometry(geometry): Update geometry, clear surfaces cache
     - get_sampler_for_mask(mask): Cache samplers by mask to avoid recomputation

3. **Create MockSurfwidgetRenderer for testing**
   - Inherits from BrainRenderer
   - is_widget() returns TRUE
   - render() records calls (like MockBrainRenderer) and returns mock htmlwidget
   - Allows testing surface view module without neurosurf dependency

4. **Update RendererRegistry**
   - Do NOT auto-register surfwidget (neurosurf may not be available)
   - Add has_surfwidget() method: checks if neurosurf is available
   - Add register_surfwidget(): conditionally registers SurfwidgetRenderer

Add tests to test-fct_brain_renderer.R:
- SurfwidgetRenderer inherits from BrainRenderer

- Neuroim2Renderer$is_widget() returns FALSE
