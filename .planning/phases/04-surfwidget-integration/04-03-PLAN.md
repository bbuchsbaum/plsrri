**Server Changes (explore_server)**

1. **Track active tab**
   ```r
   # Add to local_rv
   local_rv <- reactiveValues(
     selected_lv = 1L,
     active_tab = "volume"  # NEW: Track which tab is active
   )
   ```

2. **Observe tab changes with WebGL cleanup**
   ```r
   # Update active_tab when user switches, dispose WebGL resources
   observeEvent(input$view_tabs, {
     previous_tab <- local_rv$active_tab
     local_rv$active_tab <- input$view_tabs

     # CRITICAL: Dispose WebGL resources when switching away from Surface tab
     if (previous_tab == "surface" && input$view_tabs != "surface") {
       if (!is.null(surface_info) && !is.null(surface_info$dispose)) {
         surface_info$dispose()
       }
     }
   })
   ```

3. **Create active_tab reactive for child modules**
   ```r
   active_tab <- reactive({
     local_rv$active_tab
   })
   ```

4. **Initialize surface viewer module**
   ```r
   # Surface viewer (after brain_viewer initialization)
   surface_info <- surface_viewer_server("surface", result_rv, filters)
   ```

5. **Lazy tab rendering**
   Per CONTEXT.md: "Active tab only - other tab updates when switched to"

   The surface_viewer_server should handle its own lazy rendering internally using:
   ```r
   # Inside surface_viewer_server, add req for active tab
   output$surface_lh <- renderSurfwidget({
     # Only render when surface tab is active
     # This is handled by Shiny's built-in tab behavior
     req(result_rv())
     ...
   })
   ```

   Shiny's tabsetPanel naturally handles this - inactive tabs don't re-render until shown.

6. **Ensure filters are shared**
   Both brain_viewer_server and surface_viewer_server receive the same `filters` object from filter_bar_server. This ensures:
   - Same LV selection affects both
   - Same threshold affects both
   - Same what (bsr/salience) affects both

Add data-test attributes:

- explore-surface-tab on Surface tabPanel
