  <action>
Create new file inst/shiny/R/fct_brain_renderer.R with:

1. **BrainRenderer** abstract base class:
   - `render(result, lv, what, threshold, view, ...)` method that calls `stop("BrainRenderer$render() is abstract - implement in subclass", call. = FALSE)`
   - Document the method signature with roxygen2 comments

2. **Neuroim2Renderer** implementation:
   - Inherits from BrainRenderer
   - `render()` method delegates to `plsrri::plot_brain(result, lv = lv, what = what, threshold = threshold, view = view, ...)`
   - Handles the `along` parameter passed via `...`
   - Returns the ggplot object from plot_brain()

3. **MockBrainRenderer** for testing:
   - Inherits from BrainRenderer
   - `render_calls` field (list) to record all render() invocations
   - `initialize()` sets `self$render_calls <- list()`
   - `render()` method appends call details to `render_calls` and returns minimal ggplot: `ggplot2::ggplot() + ggplot2::ggtitle(paste("Mock LV", lv))`
   - `reset_calls()` method to clear recorded calls

4. **RendererRegistry** factory:
   - `renderers` field (list) to store registered renderers
   - `initialize()` creates empty list and registers Neuroim2Renderer as "neuroim2" by default
   - `register(name, renderer)` adds renderer, validates with `stopifnot(inherits(renderer, "BrainRenderer"))`
   - `get(name)` returns renderer or stops with error if not found
   - `list_available()` returns names(self$renderers)

Follow existing R6 patterns from inst/shiny/R/state.R:
- Use `invisible(self)` for method chaining
- Document with roxygen2 @field and @description tags
- Place all classes in single file

Use rlang::abort() for errors (consistent with research recommendation).
